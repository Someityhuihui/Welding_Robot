# 焊缝识别与轨迹规划 - 复现思路

目标：复现 `a_best_welding-robot` 核心算法，在 ROS 仿真中演示从点云到机器人焊接的完整流程。

---

## 一、算法核心流程

```
输入: 工件点云 (.pcd)
    ↓
Step 1: 点云预处理
    - VoxelGrid 下采样
    ↓
Step 2: 法向量计算
    - KdTree 半径搜索邻域点
    - 构建协方差矩阵 → SVD 分解
    - 最小特征值对应的特征向量即为法向量
    - 根据相机位置统一法向量方向（指向相机）
    ↓
Step 3: 焊缝候选区域提取
    - 计算每个点邻域法向量方向方差 (PointVariance)
    - 计算方差的方差作为描述子 (PointDescriptor)
    - 阈值筛选 → 保留高方差点（边缘/焊缝区域）
    ↓
Step 4: 聚类筛选焊缝
    - EuclideanClusterExtraction 欧氏聚类
    - 保留目标焊缝簇
    ↓
Step 5: 边界提取
    - BoundaryEstimation 提取焊缝边界点云
    ↓
Step 6: 路径点生成
    - 对每个边界点，找邻域内距离最远的两点
    - 验证两点连线与切线夹角≈90°，确认是焊缝两侧
    - 取两点中点作为焊接路径点
    ↓
Step 7: 路径点去噪、合并、排序
    - 邻域平均去噪 (DownSample_DeleteNoisePoint)
    - VoxelGrid 合并近点 (Merge_NearPoints)
    - 从几何中心出发，双向追踪排序 (Order_PathPoints_Cloud)
    ↓
Step 8: 焊枪姿态计算 (6DOF)
    - 切除头尾各3个点
    - Z轴 = 焊缝表面法向量
    - Y轴由切线方向计算
    - X轴 = Y × Z
    - X轴移动平均平滑
    - 重新正交化 Y轴 = Z × X
    - 旋转矩阵 → 四元数
    ↓
输出: 6DOF 焊接轨迹 (x,y,z,qx,qy,qz,qw)
```

## 二、关键算法详解（附源代码定位）

### 2.1 法向量计算 (PCA/SVD)

**源代码**：`seam_location.cpp:305-448`  
**函数**：`PointNormal_Computation()` / `Pointnormal_Direction_Unify()`

对于点云中的每个点 $p_i$：
1. KdTree 找半径 $r$ 内的邻域点集合 $N(p_i)$
2. 计算邻域质心：$\bar{p} = \frac{1}{|N|} \sum_{p_j \in N} p_j$
3. 构建协方差矩阵：$C = \sum_{p_j \in N} (p_j - \bar{p})(p_j - \bar{p})^T$
4. SVD 分解：$C = U \Sigma V^T$
5. 最小特征值对应的特征向量即为法向量 $n_i$（取 `U` 的第3列）

**方向统一**：计算从点到相机位置的向量 $v = p_{cam} - p_i$，若 $n_i \cdot v < 0$（夹角>90°），则 $n_i = -n_i$。

---

### 2.2 焊缝检测 (PointDescriptor)

**源代码**：`seam_location.cpp:450-750`  
**函数**：`Delete_SmoothChange_Plane()`

**核心思想**：焊缝区域法向量变化剧烈，平面区域法向量一致。

1. **PointVariance**（法向量方向方差）：
   - 对每个点 $p_i$，在其邻域内计算所有法向量与中心法向量的夹角余弦方差
   - 方差小=平面；方差大=边缘

2. **PointDescriptor**（方差的方差）：
   - 对每个点 $p_i$，在其邻域内计算所有点 PointVariance 的方差
   - 即"方差的方差"，检测局部几何变化

3. **阈值筛选**：
   - 原始代码：人工输入 `cin >> screen_threshold`，交互确认
   - **复现建议**：改用自适应阈值（如 Otsu 或百分位数），去除人工交互

---

### 2.3 边界提取

**源代码**：`motion_planning.cpp:640-720`  
**函数**：`Output_Boundary_SeamCloud()` / `Extract_Seam_edge()`

1. 对焊缝点云重新做法向量估计（`NormalEstimation`，半径 0.005）
2. `BoundaryEstimation` 提取边界点（半径 0.01）
3. 保留 `boundary_point == 1` 的点

**注意**：原始代码中 `Delete_noiseBoundary()`（聚类去噪）在调用处被注释掉了，实际没有去噪。

---

### 2.4 路径点生成

**源代码**：`motion_planning.cpp:1100-1180` / `800-890`  
**函数**：`Compute_All_PathPoints()` → `Find_relevantPoint_onTheOherCurve()` → `Compute_Single_PathPoint()`

对每个边界点 $p_i$：
1. 半径搜索邻域点（半径从 0.001 开始递增，直到找到足够点或达到 0.05）
2. 在邻域点中找到距离最远的两点 $p_a, p_b$
3. 验证：连接 $p_a p_b$ 的向量（切线方向）与 $p_i$ 到邻域内其他点的向量夹角是否 ≈ 90°（误差阈值 5°）
4. 若验证通过，路径点 = $(p_a + p_b) / 2$

**本质**：焊缝是两个工件表面的交线，路径点就是交线的中心点。

---

### 2.5 路径点去噪、合并、排序

**源代码**：`motion_planning.cpp:850-1130`  
**函数**：`DownSample_DeleteNoisePoint()` → `Merge_NearPoints()` → `Order_PathPoints_Cloud()`

1. **去噪** (`DownSample_DeleteNoisePoint`)：
   - 对每个路径点，半径搜索邻域，用邻域点平均位置替代原位置
   - 迭代 50 次（`loop_max = 50`），消除噪声

2. **合并近点** (`Merge_NearPoints`)：
   - VoxelGrid 下采样，将距离特别近的点合并为一个点

3. **排序** (`Order_PathPoints_Cloud`)：
   - 找到几何中心点作为起点（`Compute_Segment_GeometryCenter`）
   - 从起点出发，向两个方向追踪：
     - 每次找半径邻域内的点
     - 选择方向与上一段最一致（夹角<90°）的点作为下一点
     - 直到无法继续
   - 合并两个方向的序列：`half1(反转) + center + half2`

---

### 2.6 焊枪姿态计算 (6DOF)

**源代码**：`transformation.cpp:1268-1500`  
**函数**：`orientation_computation()` → `Moveit_Pose_generation()` / `URx_Pose_generation()`

**注意**：实际实现比简单描述复杂，有多层坐标变换和修正。

1. **切除头尾**：切除头尾各 3 个路径点（`pathpoint_cut_head_tail`）

2. **计算坐标轴** (`orientation_computation`)：
   - **Z轴** = 焊缝表面法向量（已统一方向，指向相机/工件上方）
   - **Y轴** = 由前后路径点的切线方向计算
   - **X轴** = Y × Z（叉乘）

3. **X轴平滑**：对 X轴 做移动平均滤波（窗口大小 7），消除抖动

4. **重新正交化**：
   - 重新计算 Y轴 = Z × X
   - 确保 XYZ 严格正交

5. **方向修正** (`direction_modification`)：进一步修正姿态方向

6. **转四元数** (`Moveit_Pose_generation`)：
   ```cpp
   // 构造旋转矩阵 [X Y Z]，列向量形式
   transformed_base << X.x, Y.x, Z.x,
                       X.y, Y.y, Z.y,
                       X.z, Y.z, Z.z;
   Eigen::Quaterniond rotation(transformed_base);
   ```

**注意**：原始代码同时生成了两套位姿：
- **MoveIt 位姿**：四元数表示，用于 Rviz/MoveIt
- **URx 位姿**：轴角表示，包含工具坐标系变换，用于直接控制 UR 机器人

## 六、关键技术备忘

### 6.1 PCL → Open3D 对照

| PCL (C++) | Open3D (Python) |
|-----------|-----------------|
| `pcl::PointCloud<pcl::PointXYZ>` | `o3d.geometry.PointCloud` |
| `pcl::KdTreeFLANN` | `o3d.geometry.KDTreeFlann` |
| `pcl::VoxelGrid` | `o3d.geometry.PointCloud.voxel_down_sample` |
| `pcl::EuclideanClusterExtraction` | `o3d.geometry.PointCloud.cluster_dbscan` |
| `pcl::MovingLeastSquares` | `o3d.geometry.PointCloud.estimate_normals` |

### 6.2 关键参数

| 参数 | 作用 | 建议值 | 源代码位置 |
|------|------|--------|-----------|
| `neighbor_radius` | 法向量估计邻域半径 | 0.005-0.01 m | `seam_location.cpp:305` |
| `voxel_size` | 下采样体素大小 | 0.001-0.003 m | `running_flow.cpp:320` |
| `cluster_tolerance` | 欧氏聚类距离阈值 | 0.002-0.005 m | `seam_location.cpp:762` |
| `boundary_radius` | 边界检测半径 | 0.01 m | `motion_planning.cpp:690` |
| `angle_threshold` | 路径点验证角度阈值 | 5° | `motion_planning.cpp:820` |
| `path_cut_count` | 切除头尾点数 | 3 | `transformation.cpp:1400` |
| `smooth_window` | X轴平滑窗口 | 7 (±3) | `transformation.cpp:1290` |

### 6.3 常见问题

1. **法向量方向不一致**：检查相机位置是否正确传入（`Pointnormal_Direction_Unify`）。
2. **焊缝检测不全**：降低 PointDescriptor 阈值或增大 neighbor_radius。
3. **路径点跳变**：增大排序阶段的 radius 或加入曲率约束。
4. **姿态奇异**：检查 XYZ 是否严格正交，必要时重新正交化（`orientation_computation` 最后一步）。
5. **MoveIt 规划失败**：轨迹点在工作空间外或碰撞，加 workspace 约束。
